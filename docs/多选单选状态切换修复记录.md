# 多选单选状态切换问题修复记录

## 问题描述

用户反馈：执行多选操作后，再进行单选操作时，画布中的元素会"乱飞"（位置错乱）。

## 问题根源分析

### 1. Fabric.js 的 activeSelection 坐标机制

Fabric.js 在处理多选时，会创建一个 `activeSelection` 对象作为临时组。关键问题在于：

- 当多个对象被选中形成 `activeSelection` 时，组内对象的 `left/top` 属性会变成**相对于组的坐标**
- 而不是画布的绝对坐标

### 2. 多选修改后状态未正确清理

当用户多选拖动完成后：

1. `onObjectModified` 事件触发
2. 代码计算并更新了 store 中的绝对坐标
3. 但 `activeSelection` 组仍然存在于画布上
4. 此时如果 `render` 方法被调用，会尝试更新组内对象的坐标
5. 这些坐标设置会变成相对于组的坐标，导致元素位置错误

### 3. 缺少多选状态管理

store 中只有 `activeElementId` 用于单选状态，没有 `selectedElementIds` 来跟踪多选状态，导致：

- 多选和单选的状态切换不清晰
- 无法正确恢复多选状态
